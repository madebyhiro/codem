Installation of Codem

Install a prepackaged, standalone version

Both the Scheduler and the Transcoder are available as standalone packages. These are experimental, alpha-stage installers. We hope you, the users, can provide us with feedback about the installation proces. Please contact us at hello@madebyhiro.com.


Scheduler

The Scheduler packages contain ruby 2.1.1, sqlite and all necessary gems to run the Scheduler. The Scheduler will be installed in /opt/codem-scheduler.

Installation from scratch

Installing Codem is a three step process.

  1. Install the prerequisites
  2. Install the Transcoder
  3. Install the Scheduler

1. Prerequisites

Packages listed here are required to run the Codem software suite.

  1. ruby >= 1.9.2
  2. rails >= 3.0
  3. node js
  4. npm
  5. ffmpeg
  6. sqlite

git is required if you want to work on the source code of Codem.

2. Installing the Transcoder

  1. Preferred way: install with npm {% highlight bash %} npm install codem-transcode {% endhighlight %}
  2. Start the server (depending on where npm installs your stuff) {% highlight bash %} /PATH/TO/TRANSCODER/bin/codem-transcode {% endhighlight %}

For more information, see the Transcoder README

3. Installing the Scheduler

  1. Either clone the git repository or download a packaged archive. {% highlight bash %} git clone git://github.com/madebyhiro/codem-schedule.git && cd codem-schedule {% endhighlight %}
  2. Install the required gems. {% highlight bash %} gem i bundler bundle install {% endhighlight %}
  3. Run the installer rake task, which will setup the application. {% highlight bash %} bundle exec rake codem:install {% endhighlight %}
  4. Use cron or a similar tool to trigger the status update task. For example, to run the task every 2 minutes, use: {% highlight bash %} */2 * * * * curl -s http://localhost:3000/api/schedule {% endhighlight %}
  5. Start the server {% highlight bash %} bundle exec rails server {% endhighlight %}

For more information, see the Scheduler README
